QUALI'MES - Logiciel MES

Manufacturing Execution System

Compétence 4 : Gérer des données de l'information
  • Collecte de données : Scrutation des machines via des box
  • Suivi en temps réel : Ordres de fabrication (OF)
  • Traçabilité complète : De la matière première au produit fini
  • Tableau de bord : Visualisation des données de production
  • Communication bidirectionnelle : Envoi de paramètres aux machines
Défi principal : Résoudre les incohérences entre données box et base de données

Débug et page d'automatisation

Problématiques identifiées

  • Activation simultanée d'opérations sur une même machine
  • Doublons d'opérations sur différentes machines
  • Incohérences état système vs réalité client
  • Calculs erronés des dates d'OF

Solution développée

Page de débugage automatisé intégrée au logiciel
  • Vérification automatique de l'intégralité des opérations
  • Comparaison fichier CSV client vs base de données
  • Multithreading : vérification machines (15min → quelques secondes)
  • Interface visuelle : codes couleur pour les statuts

Système de clignotement verrine

Objectif

Développer un système de détection et d'affichage d'alertes machines

Compétence 3 : Administrer des systèmes informatiques communicants complexes

Réalisations techniques

  • Analyse de trames : Décodage des données box par sockets
  • Calculs d'alertes : Cadence, vitesse de production, affutage
  • Synchronisation : Box ↔ Base de données ↔ Interface
  • Multithreading : Optimisation des performances
  • Interface paramétrable : Configuration des seuils d'alerte

🔵 Démonstration : Clignotement bleu en temps réel

Tableau de bord avec mise à jour automatique toutes les minutes

Intégration MongoDB

Contexte

Le client possède une base MongoDB externe avec des paramètres machines

Défis techniques

  • Connexion distante : Accès via VPN client
  • Format de données : JSON → Structure relationnelle HFSQL
  • Flexibilité : Solution adaptable à d'autres clients
